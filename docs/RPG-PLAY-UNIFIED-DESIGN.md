# RPG Play: Unified System Design Document

## ğŸ¯ Executive Summary

This document outlines the design for **RPG Play**, a unified interface that seamlessly integrates three core systems:
1. **Solo RPG** (Mythic GME - scenes, threads, fate questions)
2. **Entity Creation** (Character, location, faction generation)
3. **Tables & Oracles** (Random tables, image oracles, inspiration)

### The Turning Point

**Current Problem**: The systems exist in isolation. Players must:
- Switch between separate pages (Solo RPG, Workspace, Tables)
- Manually create entities as text descriptions
- Cannot easily reference generated entities in their game
- World generation is disconnected from play

**New Vision**: A world-first, entity-driven RPG experience where:
- **Campaigns begin with world generation** (not empty scenes)
- **Entities are living, queryable objects** (not just text)
- **All tools are contextually available** (no page switching)
- **Generation and play are seamless** (create entities inline)

## ğŸ¨ UX Design Principles

### 1. **Zero Context Switching**
- **Problem**: Currently, players must switch between /solo-rpg, /workspace, /tables
- **Solution**: All tools accessible from single interface
- **Implementation**:
  - Three-panel layout with collapsible sidebars
  - Quick action buttons in center panel
  - Tools panel with tabs (Tables, Oracles, Entities, History)
  - Keyboard shortcuts for every action (âŒ˜K, âŒ˜R, âŒ˜O, âŒ˜N)

### 2. **Entity-First Design**
- **Problem**: Current system treats characters/locations as text descriptions
- **Solution**: Entities are first-class, queryable objects
- **Implementation**:
  - @ mentions (like Slack/Discord) for entities
  - Hover previews show entity details
  - Inline entity creation from random events
  - Entity relationships automatically tracked
  - Visual entity graph (optional advanced view)

### 3. **Context-Aware Intelligence**
- **Problem**: User must manually connect related content
- **Solution**: System suggests actions based on narrative context
- **Implementation**:
  - Random event detects "NPC Action" â†’ Suggests character generation
  - Mentions location â†’ Auto-links to Location entity
  - Thread progress detected â†’ Suggests marking thread complete
  - Scene uses table result â†’ Offers "Create entity from result"
  - Chaos increases â†’ Warns of increased random event probability

### 4. **Progressive Disclosure**
- **Problem**: All features visible â†’ Overwhelming for new users
- **Solution**: Core features prominent, advanced features discoverable
- **Implementation**:
  - Level 1 (Always visible): Scene manager, quick actions, narrative timeline
  - Level 2 (One click): Tables panel, oracle cards, entity creation
  - Level 3 (Discoverable): Advanced features (Mystery Matrix, Location Crafter, Fate Check alternative)
  - Collapsible panels reduce visual clutter
  - Keyboard shortcuts for power users

### 5. **Visual Hierarchy & Clarity**
- **Problem**: Information overload in complex scenes
- **Solution**: Clear visual language and spacing
- **Implementation**:
  - **Timeline events**: Different icons for each type (â“=question, ğŸ²=roll, â•=entity, ğŸ“=note)
  - **Color coding**:
    - Blue = Information (questions, notes)
    - Green = Success (expected scenes, yes answers)
    - Red = Conflict (altered scenes, no answers)
    - Orange = Neutral (table rolls, oracle cards)
    - Purple = System (random events, suggestions)
  - **Typography**:
    - Headlines: 18-20px bold
    - Body: 14-16px regular
    - Metadata: 12px with reduced opacity
  - **Spacing**: Generous whitespace, clear section divisions

### 6. **Responsive & Performant**
- **Problem**: Complex UI can feel sluggish
- **Solution**: Optimized rendering and interactions
- **Implementation**:
  - Virtual scrolling for long timelines
  - Lazy-load entity details on hover
  - Debounced search (300ms)
  - Optimistic UI updates (instant feedback)
  - Offline-first with localStorage
  - Export/import for cloud sync

### 7. **Accessibility**
- **WCAG 2.1 AA compliance**:
  - Color contrast ratios â‰¥ 4.5:1
  - Keyboard navigation for all features
  - Screen reader labels (aria-label)
  - Focus indicators
  - Alternative text for oracle images
  - Resizable text (up to 200%)

### User Experience Goals

1. **Reduced Friction**
   - Metric: Time to create and use entity
   - Current: 2+ minutes (switch pages, generate, copy description)
   - Target: < 15 seconds (inline generation, auto-linked)

2. **Increased Engagement**
   - Metric: Average session duration
   - Current: 30-45 minutes
   - Target: 60+ minutes (fewer interruptions)

3. **Better Story Continuity**
   - Metric: Entity reuse rate
   - Current: Low (entities created but rarely referenced)
   - Target: High (entities easily referenced, quick view on hover)

4. **Reduced Cognitive Load**
   - Metric: User-reported ease of use (1-10 scale)
   - Current: 6/10 (managing three separate systems)
   - Target: 9/10 (unified interface)

### Technical Goals

1. **Performance**
   - Timeline renders < 100ms for 1000 events
   - Entity search responds < 50ms
   - Page load < 2s

2. **Reliability**
   - Zero data loss (localStorage + export)
   - Offline-first functionality
   - Auto-save every 5 seconds

3. **Maintainability**
   - Modular component architecture
   - Shared state via stores
   - TypeScript for type safety
